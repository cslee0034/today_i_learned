## 오늘 배운 점

사이드 프로젝트 팀원의 공고를 올려서 인원을 구했다.  
협업해서 좋은 코드를 만드는 것이 목적이기에 JS컨벤션, Git컨벤션 등을 정하고  
코드 리뷰 시간을 갖는 등 다양한 프로젝트 초기 논의를 하였다.

```markdown
2023-02-08 협업 일지

지금까지 혼자서 공부 해왔는데 혼자 프런트와 백 둘 다를 만들다 보니 내가 혼자 만드는게 아니었다면 몰랐을 것 같은게 상당히 많았다.
서로간의 의사 소통의 중요성을 느끼게 되어 직접 팀원들을 모집하고 협업 프로젝트를 진행하기로 했다.

프로젝트의 목표: "개인 정보를 넣으면 포트폴리오 페이지를 만들어주는" 웹페이지를 만들기로 했다. 
	- 협업 경험도 쌓고 포트폴리오로도 활용 가능할것이라 생각 되어서이다.

프로젝트의 진행 방식: Agile방식
	- 현업에서도 Agile방식의 개발을 이용해 수정이 잦고 금방 변화하는 환경에 빠르고 신뢰성 있게 개발을 한다고 한다.
	- 다만 아직 TDD에 익숙하지 못한 팀원들이 있어 테스트 코드는 작성하지 않는다.

프로젝트의 진행 전략: Git flow
	- 깃 플로우를 이용해 프로토타입인 v1을 만들어 배포하고 계속해서 Agile하게 지속적 통합으로 코드를 발전시켜 나가는 것을 목표로 한다.

회의의 결과
	1. 회의는 음성 체팅(구글밋)으로 하고, 서로 연락은 Slack으로 주고 받는다.
	2. 커밋은 Git flow로 한다 - 제목은 영어로 쓰고 내용은 한글로 쓴다. 컨벤션을 지킨다.
	3. JS의 스타일 가이드를 따라 컨벤션을 공부 하면서 프로젝트에 적용하기로 했다. 클린코드를 지향한다.
	4. 주기적으로 코드 리뷰를 하기로 한다.

디자인 패턴: Client-Server(Proxy)
렌더링 방식: CSR
사용하는 기술 Stack:
	Front: React.js, Typescript
	Back: Node.js(Express), Sequelize
	커밋: Git
	상태관리: zustand	
	CSS: tailwind
	요청쿼리: reactquery
	
	Git 컨벤션: https://www.yalco.kr/@git-github-dive/7-1/
```
